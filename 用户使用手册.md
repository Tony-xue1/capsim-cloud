## 二、DeepSeek API 连接机制

### 2.1 连接方式

网站通过**服务端代理**方式调用 DeepSeek API，流程如下：

```
用户浏览器 → Railway 服务器(server.js) → DeepSeek API → 返回分析结果
```

- **API 地址**：`https://api.deepseek.com/v1/chat/completions`
- **调用模型**：`deepseek-chat`
- **API Key 存储**：环境变量 `DEEPSEEK_API_KEY`（存储在 Railway 平台，不暴露在前端）

### 2.2 代码位置

在 `server.js` 第 491-521 行的 `/api/analyze` 接口中实现：

```javascript
const response = await fetch(`${apiBase}/chat/completions`, {
  method: "POST",
  headers: { "Content-Type": "application/json", "Authorization": `Bearer ${apiKey}` },
  body: JSON.stringify({ model, messages: [{ role: "user", content: prompt }], temperature: 0.3, max_tokens: 4000 }),
});
```

### 2.3 API Key 安全性

| 项目 | 说明 |
|------|------|
| Key 存储位置 | Railway 环境变量（服务端） |
| 前端是否可见 | ❌ 不可见，前端只调用 `/api/analyze` |
| Key 传输方式 | 服务端直接发送给 DeepSeek，不经过用户浏览器 |
| 修改 Key 方法 | 登录 Railway → 项目 → Variables → 修改 `DEEPSEEK_API_KEY` |

### 2.4 如何更换 API Key

1. 登录 [Railway](https://railway.app)
2. 进入 `capsim-cloud` 项目
3. 点击服务 → **Variables** 标签
4. 修改 `DEEPSEEK_API_KEY` 的值
5. 点击 **Redeploy** 重新部署

### 3.2 AI 分析三阶段说明

```
阶段一：识别产品现状
  → 从 PDF 报告中提取我方产品参数（性能、尺寸、MTBF、价格等）
  → 与官方理想点对比，计算偏差

阶段二：问题诊断
  → 财务健康度分析（利润率、ROE、现金流等）
  → 竞争力差距分析

阶段三：决策参数建议
  → R&D 研发决策（目标性能/尺寸/MTBF）
  → Marketing 营销决策（定价、促销、销售预算）
  → Production 生产决策（产量、扩产、自动化）
  → Finance 财务决策（融资、股息）
  → TQM 质量决策（各项投入建议）
```

## 四、用户使用指南

### 4.1 普通用户使用流程

1. **打开网站**：访问 https://capsim-cloud-production.up.railway.app
2. **登录**：输入管理员分配的用户名和密码
3. **配置参数**：
   - 填写队伍名称（如 Andrews、Baldwin）
   - 选择当前轮次（第0-8轮）
   - 选择战略方向（可选）
4. **上传报告**：点击上传区，选择 Capsim Courier 报告 PDF
5. **开始分析**：点击「⚡ 开始三阶段 AI 分析」按钮
6. **查看结果**：等待 AI 返回分析报告（约30-60秒）
7. **复制结果**：点击「📋 复制结果」按钮，粘贴到需要的地方

### 4.2 管理员操作指南

#### 添加新用户
1. 登录后点击导航栏「👥 用户管理」
2. 点击「+ 添加用户」
3. 填写用户名、密码（至少6位）、选择角色（成员/管理员）
4. 点击「创建用户」

#### 删除用户
1. 在用户列表中找到目标用户
2. 点击「删除」按钮
3. 确认删除（不能删除自己，不能删除最后一个管理员）

#### 修改管理员密码
1. 点击导航栏「修改密码」
2. 输入旧密码和新密码（至少6位）
3. 确认修改

### 4.3 注意事项

- PDF 必须包含**可选择文字**，扫描件（图片PDF）无法解析
- PDF 文件大小限制 **50MB**
- 登录令牌有效期 **7天**，过期需重新登录
- AI 分析需要消耗 DeepSeek API 额度，请合理使用

### 5.3 公网安全设置建议

#### 建议立即执行（高优先级）

1. **修改默认管理员密码**
   - 登录网站 → 修改密码 → 设置强密码（字母+数字+符号，8位以上）

2. **限制 CORS 来源**
   ```javascript
   // server.js 第169行改为：
   app.use(cors({ origin: "https://capsim-cloud-production.up.railway.app" }));
   ```

3. **添加速率限制**
   ```bash
   npm install express-rate-limit
   ```
   ```javascript
   // 在 server.js 添加：
   import rateLimit from "express-rate-limit";
   const limiter = rateLimit({ windowMs: 15*60*1000, max: 50 });
   app.use("/api/", limiter);
   ```

4. **添加安全头**
   ```bash
   npm install helmet
   ```
   ```javascript
   // 在 server.js 添加：
   import helmet from "helmet";
   app.use(helmet());
   ```

#### 中期改进建议

5. **接入 PostgreSQL 数据库**（Railway 内置支持）
   - 解决 Railway 重启后用户数据丢失问题

6. **添加登录日志**
   - 记录谁在什么时间登录、分析了什么报告

## 七、常见问题

### Q1：忘记管理员密码怎么办？
A：登录 Railway → 修改变量 `ADMIN_PASSWORD` → Redeploy。重启后密码会重置为新值。

### Q2：上传 PDF 提示"文字内容过少"？
A：PDF 可能是扫描件（图片）。需要使用 OCR 工具转换为可选择文字的 PDF。

### Q3：AI 分析返回"未配置 API Key"？
A：检查 Railway 环境变量 `DEEPSEEK_API_KEY` 是否正确设置。也可能是 API 额度用完。

### Q4：网站打不开？
A：检查 Railway 服务是否正常运行。访问 `https://capsim-cloud-production.up.railway.app/api/health` 查看健康状态。

### Q5：Railway 重启后用户数据丢失？
A：当前使用 JSON 文件存储，Railway 重启会重置。建议接入 PostgreSQL 数据库（见5.3节）。

### Q6：境内访问速度慢？
A：Railway 服务器在海外。如需更快速度，可考虑：
   - 使用国内云服务器（阿里云/腾讯云）+ 已备案域名
   - 使用 Cloudflare CDN 加速
